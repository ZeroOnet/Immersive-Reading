## A. 语音施法 / 朗读触发（对标：Spell Voice Unlock、Read to Restore）

### ⭐ In Verbis Virtus — 对着麦克风念咒语施法
- **是什么**：第一人称解谜游戏，玩家**真的念出咒语**（自创语言 Maha'ki）来施放魔法、解谜、战斗。
- **技术启示**：用 **PocketSphinx (CMUSphinx) 离线关键词/音素匹配**，不做完整句 ASR——**这正是"逐词点亮"该用的稳妥做法**：只匹配目标词，而不是听写全句。
- 链接：[Steam](https://store.steampowered.com/app/242840/In_Verbis_Virtus/) · [Wikipedia](https://en.wikipedia.org/wiki/In_Verbis_Virtus)

### ⭐ 微软 Reading Coach / Reading Progress — 朗读逐词高亮 + 纠错
- **是什么**：学生朗读课文，speech-to-text **实时分析流利度、逐词高亮、标出读错的 5 个难词、给准确率/语速评分**。
- **技术启示**：**这就是你"逐词点亮"的生产级形态**，而且是阅读教育场景——直接是扇贝口语/跟读能力的对标物。路演时可引用为"这套能力业界已成熟"。
- 链接：[coach.microsoft.com](https://coach.microsoft.com/)

### Google Read Along（原 Bolo）— 儿童朗读随读高亮
- **是什么**：儿童读绘本，**文字随朗读高亮**，读不出点词即发音。已出网页 beta。
- **技术启示**：移动端"朗读驱动阅读"的轻量落地范例。

## B. 文字即环境 / 动态排版互动小说（对标：文字桥、兔子洞、沉浸阅读壳）

### ⭐ Device 6（Simogo）— 文字本身就是地图
- **是什么**：文字排版**跟着主角移动转向、分叉**，文字即走廊/楼梯/空间，配合声画。交互叙事排版的天花板。
- **技术启示**：你"文字拼成桥/路/兔子洞"的灵感原型；证明"文字当场景"可以撑起整个体验。
- 链接：[Simogo 作品页](https://simogo.com/work/device-6/) · [Wikipedia](https://en.wikipedia.org/wiki/Device_6)

### Pry（Tender Claws）— 可触摸的文字空间
- 把文字变成能扒开/挤压的空间，读者"撬开"文字进入潜意识。多模态触觉阅读参考。

### The Boat（SBS）— WebGL 互动图像小说
- 前面已拆过实现原理（Three.js + GLSL + 滚动当播放头）。沉浸阅读壳的视觉标杆。

## C. 文字即谜题（对标：抽词组成谜底、Evidence Board、解码）

### ⭐ Blackbar — 猜出被审查遮盖的词
- **是什么**：通信里关键词被黑条遮住，玩家**根据上下文推理/解出被遮的词**才能推进剧情。纯文字解谜。
- **技术启示**：**最贴你"被抽出的词组成谜底/解锁条件"**；也印证"查词/读懂=解谜"的可玩性。
- 姊妹作 **Grayout**（反乌托邦填词解谜）。
- 参考：[ResetEra 介绍](https://www.resetera.com/threads/blackbar-grayout-a-dystopian-duo-of-gripping-interactive-fiction-word-games.2434/)

## D. 文字显影 / 擦除（对标：血字 RACHE、月光阅读、窗上雾字）

核心技术统一是 Canvas `globalCompositeOperation = 'destination-out'`（盖一层、擦掉露出底层）。现成库直接抄：

- **wc-scratch / ScratchCard.js**（sebastianwachter）— Web Component，设 `percentRequired` 到阈值触发"显影完成"回调。[GitHub](https://github.com/sebastianwachter/ScratchCard)
- **ScratchIt**（jQuery）、**jQuery Eraser** — 老牌实现。
- 教程：[Tuts+ 纯 JS scratch card](https://webdesign.tutsplus.com/how-to-create-a-scratch-card-effect-in-vanilla-javascript--cms-108922t)、[CSS-Script scratch-it](https://www.cssscript.com/scratch-ticket-javascript-canvas/)
- **技术启示**：和我们 spike 计划里 P3 同原理；可以直接用库或自写 ~30 行。**风险最低的模块。**

## E. 声控 / 音量 / 喊叫（对标：敲三下开门、音量施法、喊出名字）

### ⭐ Yasuhati / Don't Stop Eighth Note（Scream Go Hero）— 纯靠音量控制
- **是什么**：**小声走、大声跳、尖叫高跳**，屏幕底部有"麦克风灵敏度条"。中日社交网络爆火。
- **技术启示**：①**只测音量阈值、不做声音识别**——和我 `knock-mic.html` spike 完全同思路，最稳；②那条**灵敏度条**就是现场兜底（嘈杂环境可调），值得照抄；③证明声控玩法的传播力。
- 链接：[App Store](https://apps.apple.com/us/app/yasuhati-official/id1215691000) · [Google Play](https://play.google.com/store/apps/details?id=jp.ne.freem.YASUHATI)

## I. 国际深挖 —— 更贴脸的产品级案例（最该看这几个）

### ⭐⭐ Novel Effect — 朗读时，故事世界实时回应（你的多模态核心，已是获奖消费品）
- **是什么**：你**照着绘本朗读**，App 用语音识别**跟着你读到哪、就在那一刻播放对应的配乐 / 音效 / 角色声**。支持上百本畅销童书，iOS/Android/桌面都有。
- **为什么关键**：这几乎就是你"用户朗读 → 文字/世界回应"的**已上线、获奖、规模化**版本（只是它回应的是声音，你要回应的是文字魔法）。**直接证明你的 hero idea 成立且有人愿意付费。**
- **技术启示**：它也是"跟读到哪、就触发哪"，和你"逐词点亮"同构；用**词序匹配 + 容错**，不强求完整 ASR。
- 链接：[noveleffect.com](https://noveleffect.com/) · [App Store](https://apps.apple.com/us/app/novel-effect-read-aloud-books/id1057374139)

### ⭐⭐ Pottermore / Wizarding World — 整本 IP 小说 + 阅读中施法/收集/酿药
- **是什么**：官方做过的"互动阅读"——逐章跟读《魔法石》，**收集魔药材料、卡片、金加隆**；在决斗里**敲/点出一串字母来施法**；按精确步骤**酿造魔药**；章节里有可交互的 "Moment" 插画。后改版下线了互动部分。
- **为什么关键**：这是**版权方亲自做的"读一本 IP 原著、阅读中穿插魔法互动"**——和你的概念几乎重合。也直接印证我们文档里的版权判断：**强 IP 自己都收了，你必须抽象成 genre。**
- **可借走**："决斗时敲出字母施法" = 文字即施法输入；"按步骤酿药" = 操作序列解谜。
- 链接：[Wizarding World Digital（维基）](https://en.wikipedia.org/wiki/Wizarding_World_Digital)

### ⭐ inkle — 经典名著做成互动叙事（学院派背书）
- **Frankenstein: Interactive**（Dave Morris × Profile Books）——玛丽·雪莱原著的选择式互动版，被 Salon 评"最好的互动小说"，"像一本自己会写的书"。
- **80 Days**——儒勒·凡尔纳《环游世界八十天》改编，50 万字脚本，**TIME 2014 年度游戏 + 4 项 BAFTA 提名**。
- **为什么关键**：证明"**公版经典名著 → 互动叙事**"这条路能做到获奖级；你要叠的是"文字特效 + 学习行为"，叙事框架有现成范式。
- 链接：[inklestudios.com](https://www.inklestudios.com/) · [Frankenstein](https://apps.apple.com/us/app/frankenstein-interactive/id516047066)

### ⭐ Unmemory —— "能读的游戏，能玩的书"（边读边解谜/解码）
- **是什么**：8 章、滚动式阅读，**文字本身就是玩法**，剧情里嵌入大量**密码/解码谜题**，编辑设计精致（90 年代 noir 风）。
- **为什么关键**：**最贴你"文字即谜题 / 摩斯/凯撒解码"**那条线的成品形态；展示了"阅读流 + 解谜"如何无缝编织。
- 链接：[Steam](https://store.steampowered.com/app/1388860/UNMEMORY/) · [App Store](https://apps.apple.com/us/app/unmemory/id1525188600)
- 同类实体书：**The Cypher Files / Journal 29**——每页解谜、线上提交答案拿钥匙推进。[thecypherfiles.com](https://thecypherfiles.com/)

### ⭐ Type:Rider —— 角色在"文字/字母"里穿行
- **是什么**：你操控一个**冒号 `:`** 滚过十个由真实字体（Garamond、Baskerville、Helvetica…）构成的世界，**在字母间跳跃、收集字母**，顺带学字体史。
- **为什么关键**：**正是你"小人从文字桥上走过 / 沿字母脚印前进 / 阅读进度=角色在文字世界移动"的成品证明。**
- 链接：[Steam](https://store.steampowered.com/app/258890/TypeRider/) · [Wikipedia](https://en.wikipedia.org/wiki/Type:Rider)

### ⭐ Between Page and Screen —— 文字重排成形状（AR/电子文学经典）
- **是什么**：一本"没有字"的诗集，每页是黑白几何码，**对着摄像头**才浮现 3D 动态文字——**文字在空间里移动、重排成形状（具象诗）**。
- **为什么关键**：你"文字拼成物件/形状""文字飞出重组"的艺术化先例；也是多模态（摄像头）阅读的范例。
- 链接：[betweenpageandscreen.com](https://www.betweenpageandscreen.com/) · [Wikipedia](https://en.wikipedia.org/wiki/Between_Page_and_Screen)
