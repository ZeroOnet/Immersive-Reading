# Magic Text Reading — 技术储备

> 面向 2026 扇贝黑客松（产品创新赛道）。本文档是动手前的技术地基：概念定位、对评分表的判断、模块×技术原语可行性矩阵、共享引擎架构、关键 API 选型、风险与兜底、以及 AI 的真正落点。
>
> 配套的高风险原语验证见 `../spikes/`（可直接用浏览器打开，手机也能测）。

---

## 0. 一句话概念

**不是给阅读页加特效，而是把"看、查、读、划、敲、喊"等英文学习行为，变成推进原著剧情的机关——文字会回应用户。**

最终交付：一个**前端展厅式网页**，纵向滚动，嵌入 4–5 个**可真实交互**的 Magic Text 模块，用于路演拉票。每个模块 = 介绍区 + 交互演示区。

---

## 1. 对评分表的冷静判断（决定打法）

评分：**AI创新性 35% · 实用性 25% · 完成度 20% · 商业/业务价值 20%**。

| 维度 | 现方案表现 | 说明 |
|---|---|---|
| 实用性 25% | ⭐⭐⭐⭐⭐ | 查词/划线/生词本/朗读全是扇贝现成行为，目标用户清晰 |
| 完成度 20% | ⭐⭐⭐⭐⭐ | 纯前端、无后端、可控，路演只需一个 URL |
| 商业价值 20% | ⭐⭐⭐⭐ | IP 原著专区 / 会员权益 / 完读率，叙事顺 |
| **AI创新性 35%** | ⭐⭐ ⚠️ | **最大风险**：当前 95% 是 CSS/Canvas/WebAudio 触发的动画，几乎不含 AI |

**结论**：这个创意在"产品创新/实用/完成度"上很稳，但**权重最高的 AI创新性会拖后腿**——评委按表打分时，一堆"点击触发动画"会被判为"特效合集"，不是"AI 创新"。ChatGPT 那轮没点破这一点。**必须补 AI。**

## 2. 补 AI：把"特效合集"升级成"AI 引擎"（决定性）

### 2.1 Spellbook Compiler（核心杀招）

一个 LLM 流水线：**输入任意公版原著章节 → 自动产出"魔法脚本"JSON → 驱动前端引擎渲染互动关卡。**

LLM 要做的事：
1. **判定文本气质** → 归类 genre（侦探 / 奇幻 / 科幻 / 哥特 / 海洋 / 治愈…）→ 匹配特效模板。
2. **抽取交互锚点**：哪几个词作为"线索词/可查词"、哪句作为"咒语句"、谜底是什么、解锁条件。
3. **生成衍生内容**：查词释义、morse 信号、cipher、"被风吹走后保留的关键词摘要"。
4. **输出结构化 JSON**（schema 见 §5），前端零改动直接消费。

为什么这是杀招：
- **AI创新性 35% 拉满**：核心是"用 LLM 把文学文本编译成可玩的学习关卡"，这是真 AI、有想象力。
- **商业价值 20% 同时拉满**：内容不再靠人手编排 → **任意一本书都能自动生成** → 真正可规模化、可上线、可做成 IP 原著专区的生产管线。
- **路演杀手锏**：现场粘贴一段《呼啸山庄》，~10 秒生成一套互动阅读关卡，全场直接看懂"这是个引擎，不是几个 demo"。

> 重新定位一句话：**Magic Text Reading 是一个把任意英文原著自动编译成多模态互动阅读关卡的 AI 引擎，那些神奇文字效果只是它的渲染层。**

### 2.2 语音施法（第二个真 AI 落点）

"念咒语逐词点亮"本身就是**语音识别 / 口语评测**，天然复用扇贝口语能力，是多模态 AI 的体面落点。详见 §4 的 P4。

实现层：黑客松 Demo 用 **Web Speech API** 即可；产品化叙事再提扇贝自有 ASR/口语评分。

## 3. 共享引擎架构（一套底座，跑所有模块）

```
Gallery 页（Lenis + GSAP ScrollTrigger，纵向滚动叙事）
│
├─ <ModuleSection> 通用容器：左讲解 / 右交互区 + 「demo mode 兜底」按钮
│
├─ 渲染层 Renderers（按模块复用）
│   ├─ DOMTextRenderer   可点击词 / 高亮 / 证据板          ← 查词类
│   ├─ ParticleText      文字→粒子：坍塌/桥/风/引力/兔子洞   ← 形变类（spike 3）
│   ├─ EraseReveal       canvas destination-out 擦除显影     ← 血字类
│   └─ AuroraShader      WebGL 片元氛围背景（已在 Magical Book 备好）
│
├─ 输入层 Inputs（多模态，全部带 fallback）
│   ├─ PointerInput      点击 / 拖动 / 长按划线
│   ├─ SpeechInput       Web Speech 识别 → 逐词匹配点亮      ← spike 1
│   └─ MicInput          getUserMedia + Analyser RMS 峰值     ← spike 2（敲击/音量）
│
├─ 内容层
│   └─ SpellScript（JSON） ← Spellbook Compiler（LLM）产出，§5
│
└─ 引擎库：GSAP(+ScrollTrigger, SplitText) · Lenis · 原生 Canvas/WebGL/WebAudio
```

**复用红利**：`Magical Book` 项目已经把 Lenis / GSAP ScrollTrigger(scrub) / Canvas 粒子 / WebGL 着色器 / WebAudio 这套地基搭好并验证过构建——本项目的渲染层和滚动叙事可直接搬过来，省掉地基时间。

> **架构硬约定（已定）**：渲染层全部做成参数化的 `effects/` 模块（唯一真源），调试台 `lab/`
> 与成品展厅 `gallery/` 都只消费它，调试 UI 永不进成品包。详见
> [`调试与成品分离.md`](调试与成品分离.md)。`effect.schema` 一份三用：Lab 自动生成控制面板 /
> 成品参数校验 / 约束 Spellbook Compiler 的 AI 输出。

## 4. 模块 × 技术原语 可行性矩阵

风险=现场翻车概率。难度=实现工作量。所有"高"风险项必须配 demo-mode 兜底。

| # | 原语 | 支撑哪些模块 | Web API / 方案 | 难度 | 风险 | 兜底 |
|---|---|---|---|---|---|---|
| P1 | 可点击词 / 高亮 / 证据板 | Evidence Board、查词线索 | DOM `<span>` + click + SVG 连线 | 低 | 低 | — |
| P2 | **文字→粒子形变** | 坍塌成山、文字桥、风吹竖线、三星引力、兔子洞 | Canvas `measureText`→`getImageData` 采样成点 + 缓动/物理 | 中 | 中 | 静态 SVG 兜底 |
| P3 | 擦除显影 | 血字 RACHE、月光阅读、窗上雾字 | Canvas `globalCompositeOperation='destination-out'` | 低 | 低 | 一键全显 |
| P4 | **语音逐词点亮** | Spell Voice Unlock、跟读修复 | `SpeechRecognition`(webkit) + 词序匹配 | 中 | **高** | demo-mode 自动逐词亮 |
| P5 | **麦克风敲击/音量** | 敲三下开门、文字雨开花、音量施法 | `getUserMedia` + `AnalyserNode` RMS 峰值 | 中 | 中 | 「knock for me」按钮 |
| P6 | 摩斯/信号听力 | Listen to the Signal | WebAudio `OscillatorNode` 播放 + 输入校验 | 低 | 低 | 显示节奏可视化 |
| P7 | 滚动驱动叙事/转场 | 整个 gallery、文字门转场 | Lenis + GSAP ScrollTrigger `scrub` | 低 | 低 | — |
| P8 | 着色器氛围背景 | 各模块情绪底色 | WebGL fragment shader（已备） | 低 | 低 | CSS 渐变 |
| P9 | **Spellbook Compiler** | 全部（内容生成） | LLM 结构化输出（Claude）→ JSON | 中 | 中 | 预生成好的 JSON 随包 |
| P10 | **设备体感物理** | 字母雨/倾斜掉落/摇晃、《爱丽丝》梦境飘落 | `DeviceOrientation`/`DeviceMotion` + 2D 物理(重力矢量+边界+互斥堆积) | 中 | **高** | gravityAngle 滑块模拟倾斜 + Reset=摇晃 |
| P11 | **DOM 正文扰动** | 《呼啸山庄》荒原风暴正文、滚动抖动、风停显字 | DOM 词级 transform(translate/skew/scaleX)+容器 blur，**风强/滚动速度**驱动 | 中 | 低 | windIntensity 滑块 + 滚轮阵风 |
| P12 | **海上雷暴场景（合成时刻）** | 《老人与海》天空+雨+浪上浮动+雷声；**海"浪"=一簇簇水珠以随机角度/散开泼向镜头**(由小变大带拖影)；**随机 1~N 条闪电照亮天空**+劈裂文字 | 飞溅粒子(随机起点/中心角/散开，z 朝镜头放大加速+拖影 → **黏住后缓慢下滑并渐隐**，叠加混合) + 雨层 + 多条带分叉电弧 + 闪光提亮 + WebAudio 噪声(海浪/雷) + 文字裂开 | 高 | 中 | 音频需手势；`飞溅频率/散开角/迸射速度/冲镜放大` 可调；Reset=手动劈闪电 |
| P13 | **仿真纸张燃烧（噪声溶解）** | 《飘》战火烧页：有机燃烧前沿向内推进、烧穿露暗底、近火文字焦化 + 火星 + 战场音效 | **逐像素噪声阈值溶解**：每像素燃烧次序 D = (1-边缘距) + fbm 噪声扰动；burn 扫过 D → 烧穿(透明)/余烬鳞带(白热→橙→焦黑)/焦黄纸/完好纸（ImageData，burn 变才重算，离屏缓存逐帧 drawImage）+ 火星 + WebAudio | 中 | 中 | burn 滑块=进度；音频需手势；Reset 重生噪声形状 |
| P14 | **连续音量驱动的累积显现 / 填充** | 《魔戒》召唤显现金铭：大声读出铭文→实时变清晰；持续召唤累积金色填充（像指纹录入）直至绽放 | `getUserMedia`+`AnalyserNode` **连续 RMS**（非峰值）→ 实时辉光/清晰度 + 累积 `fill`；环形文字沿圆周排布、金色按前沿推进 + 流动微光 + 雨痕随完成消退 | 中 | **高** | 麦克风需安全上下文；fill 滑块预览 + **按住画面**无麦兜底；Reset 重新冲淡 |
| P15 | **天气粒子（雨/雪）** | 通用下雨/下雪层，可独立用、也可叠进任何场景 | canvas 粒子带**景深 parallax**(z→大小/速度/透明度)；雨=带风斜线、雪=柔点+横摆；环绕回收 | 低 | 低 | 独立效果 `precipitation`，rain/snow 切换 |

**结论**：真正不确定、必须先验证的是 **P4（语音）、P5（麦克风）、P2（粒子形变）、P10（体感物理）**。前三个是 spike（§6），P10 已直接做成 lab 效果 `letterRain`（§6）。P9 是 AI 核心、风险在 prompt 工程，可离线预生成兜底（现场不依赖实时网络）。

**P10 的坑（务必记住）**：① iOS 13+ 的 `DeviceOrientationEvent.requestPermission()` **必须在用户手势里调用**，且页面要 **HTTPS**——LAN 上的 `http://10.x` 在 iPhone 上拿不到体感（同 mic/speech 的安全上下文限制）；② 桌面**没有传感器**，所以效果内置「滑块模拟倾斜 + Reset=摇晃」兜底，desktop 也能调；③ 任意重力方向的"堆积"用边界碰撞 + O(n²) 互斥近似（字母数 ≤90），不是刚体物理，但视觉够用。

> 另一个《爱丽丝》效果 **Drink Me 文字变大变小**（点小瓶 → 字号/行距膨胀、个别词像家具放大、缩小的 Alice 从词间穿过）属低风险：纯 DOM/CSS scale + 角色路径动画（参照 Type:Rider 的"在字母间穿行"），无需新原语。**现已做成 lab 效果 `effects/drinkMe/`**（见 §6）。

## 5. SpellScript JSON（前端 ⇄ Compiler 的契约）

前端引擎只认这个结构；LLM 负责产出它。先把契约定死，前后端可并行。

```jsonc
{
  "source": { "title": "Wuthering Heights", "license": "public-domain", "gutenbergId": 768 },
  "genre": "gothic",                 // 决定渲染模板 / 配色 / 氛围
  "ambientHue": 210,
  "passage": "原文段落（已分句）...",
  "interactions": [
    {
      "type": "lookup-clue",         // 查词即线索
      "words": [                     // 可点击查词的锚点
        { "surface": "scarlet", "gloss": "猩红色", "isClue": true },
        { "surface": "wall", "gloss": "墙", "isClue": true }
      ],
      "unlock": { "needClues": 3, "answer": "RACHE" }
    },
    {
      "type": "spell-voice",         // 念咒语逐词点亮
      "sentence": "The hidden gate opens only to the voice that remembers.",
      "tts": true,
      "onComplete": "open-door"
    },
    {
      "type": "wind-summary",        // 被风带走 → 保留关键词（AI 摘要）
      "keepWords": ["house", "remembers", "every", "word"]
    }
  ],
  "transition": { "type": "text-door", "nextTitle": "Chapter II" }
}
```

Compiler 调用要点：用 Claude 的**结构化输出 / 工具调用**强制吐合法 JSON（schema 校验 + 失败重试），避免自由文本解析。文本一律取 **Project Gutenberg**（公版，安全）。
