## 1. 现有资产盘点（直接复用）

`src/effects/registry.ts` 里已登记 18 个 effect。其中**整页阅读皮肤**（即可塞进手机框的成品 Demo）正好覆盖四大模块：

| 模块 | 书 | 对应 effect id | 现有交互 | 路演可驱动方式 |
|---|---|---|---|---|
| ① 氛围营造 | 呼啸山庄 | `wutheringSkin` | 环境氛围（视频/风/雾，自播） | 进入视口即 mount，自动播放 |
| ① 氛围营造 | 老人与海 | `oldManSkin` | 海浪/船头氛围（自播） | 同上 |
| ① 氛围营造 | 飘 | `goneSkin` | 火光/誓言（自播） | 同上 |
| ② 氛围+互动 | 血字的研究 | `sherlockSkin` | RACHE→RACHEL→revenge 反转 | 滚动/按钮分步触发 |
| ② 氛围+互动 | 指环王 | `ringSkin`（+`ringInscription`） | 语音点亮魔戒铭文 | 滚动驱动点亮进度 +（可选）麦克风 |
| ③ WOW 奇幻 | 爱丽丝（纸牌） | `aliceSkin` | 点词→红心/菱形飞出撞散文字 | 自动/手动点词 |
| ③ WOW 奇幻 | 爱丽丝（兔子洞） | `aliceFallSkin` | 文字/物件坠落 | 「再晃一下」按钮 / 滚动驱动 |
| ④ 角色内心 | 面纱 | `veilSkin`、`veilSmokeSkin` | 迷雾散开露真意 | 滚动/滑动驱动 |
| ④ 角色内心 | 傲慢与偏见 | `prideSkin` | 揭纸条露傲慢（`peel` 0→1） | 滚动驱动 `peel` + 手动拖 |

其余为参考原语（`textParticles / letterRain / stormText / oceanStorm / pageBurn / drinkMe / precipitation / ringInscription`），门户暂不直接展示，按需作首屏 Hero 的氛围碎片。

**EffectModule 契约**（`src/effects/types.ts`，门户消费这套，不改契约）：
```ts
mount(container, params) → handle
handle.update(partial) / resize() / destroy() / getParams()
mod.defaultParams / mod.presets
```

> ⚠️ **实现期需核对的能力缺口**：方案要求「滚动自动演示」。
> 目前能被 `update()` 直接驱动进度的只有少数 effect（如 `prideSkin.peel`）。
> 其余「自动点亮 / 自动反转 / 自动坠落」需要在实现期给对应 effect **补一个可驱动入口**
> （二选一：暴露一个 0..1 进度参数走 `update()`；或加一个 `handle.play?()` 可选能力到契约）。
> 这是本项目最大的工程量与风险点，已在 §7 排期里单列。

## 3. 核心组件设计

### 3.1 PhoneFrame（`phone.ts`）

一个把任意 effect 跑在「手机里」的组件：

- **设备框**：圆角 + 黑边框 + 动态岛/刘海占位；内屏 **375×794**（effect 自带状态栏 9:41，无需门户再画）。
- **缩放适配**：内屏恒定 375×794，整框用 CSS `transform: scale()` 适配不同视口（路演投影/笔记本）。缩放只动框，不动 effect 内部布局 → 还原不失真。
- **懒挂载 / 销毁**：进入视口（IntersectionObserver，`rootMargin` 预取一屏）→ `mod.mount()`；离开较远 → `handle.destroy()`。**全程同时运行的 effect 控制在 1–2 个**，避免十几个 rAF 同时跑卡顿。
- **接口**：
  ```ts
  createPhone({ effectId, params })       // params 可来自 mod.presets[x] 或定制
  phone.el                                // 挂进 section 的 DOM
  phone.onEnter / onExit                  // 由 scroll.ts 调用做 mount/destroy
  phone.handle?                           // mount 后透出，供滚动驱动 update()
  ```

### 3.2 滚动驱动（`scroll.ts`）

- `observeMount(phone)`：IO 控制 mount/destroy。
- `observeProgress(sectionEl, cb)`：把某 section 在视口内的滚动进度算成 `0..1`，回调里 `phone.handle.update({ <progressParam>: t })`。用于 `prideSkin.peel`、魔戒点亮进度等「滚动即播放」。
- 交互型 effect（点词、点麦克风）：在手机旁放一个 **「点击体验 / Try it」按钮**，按钮触发 effect 的交互（或合成事件）；同时尽量给一个滚动自动版，保证评委不操作也看得懂。

### 3.3 data.ts（唯一文案真源）

把所有屏的标题/副文案/按钮/书单/effect 映射集中一处，section builder 只渲染、不写死文案。全部文案见 §5。

### 03 核心理念
标题：`多模态互动式阅读，不是把素材放在一起。而是让多模态和文本发生反应。`
中央公式（做成粒子/卡片汇入手机屏）：
```
文本内容 + 视觉场景 + 声音反馈 + 动态视频 + 用户行为 = 沉浸式文学体验
```
滚动逐层浮现：文字→背景视频→音效波形→粒子→用户手势→融合成发光阅读页。

### 05 模块一：氛围营造
标题：`01 氛围营造 — 把文字所在的世界，变成用户正在进入的环境。`
布局：左大文案 / 右手机 / 背景随滚动切三本书。
- 呼啸山庄：冷灰蓝荒原、风雨雾。`荒原的风不是背景。它是这本书情绪的一部分。`
- 老人与海：深蓝海面、船头、风暴。`当海浪压近屏幕，读者感受到的不只是孤独，还有搏斗。`
- 飘：火烧天空、废墟、斯嘉丽背影。`一句誓言，被火光和灰烬重新点燃。`
总结句：`模块一解决的是：让用户先"进入"作品，而不是先理解作品。`

### 06 模块二：氛围 + 互动
标题：`02 氛围营造 + 互动 — 关键意义，不是直接告诉用户，而是让用户亲手触发。`
左《血字的研究》：`视觉解谜：RACHE 的两层反转` — Step1 发现线索 / Step2 误入推理(RACHEL) / Step3 真相反转(Rache=revenge)。
`悬疑阅读的爽点，不只是知道答案。而是先被误导，再被推翻。`
右《指环王》：`声音召唤：念出咒语，点亮魔戒` — 麦克风波形 + "One Ring to rule them all..." 逐词高亮 + 铭文从左到右点亮 + 熔岩火光。
`魔法类文本不只适合被阅读。它天然适合被念出、被召唤、被点亮。`
总结句：`模块二解决的是：让用户从旁观者变成触发者。`

### 07 模块三：WOW 奇幻体验（整屏只讲爱丽丝）
标题：`03 WOW 奇幻体验 — 把不可能发生的文学想象，变成屏幕里真实发生的事件。`

  `点击一个词，纸牌人物从牌面里挥出爱心与菱形。图形飞过文本，文字被撞散。`
- 段二（兔子洞坠落）：点「再晃一下」→ 正文倾斜 → 文字像物体滑落 → 掉进底部黑白旋涡 → 留白渐多。
  `在《爱丽丝》中，荒诞不是装饰风格。荒诞就是页面本身的物理规则。`
强按钮：`Try the Rabbit-Hole`。总结句：`模块三解决的是：让阅读出现一次真正意想不到的"奇观时刻"。`

### 08 模块四：走进角色内心
标题：`04 走进角色内心世界 — 揭开一句话背后的真实情绪、谎言与潜台词。`

## 6. 待你拍板的几个设计点

1. **sticky 手机形态**：
   - (A) **每个模块各自一台 sticky 手机**（推荐）：稳、实现简单、effect 各自独立 mount/destroy。
   - (B) 单一全局 sticky 手机，滚动时切换 effect：最像 Apple，但要处理 effect 热切换/状态保持，复杂度与卡顿风险高。
2. **整屏 scroll-snap**：要不要每屏吸附（更"发布会"感，但和滚动驱动动画偶有冲突）？还是自由滚动。
3. **魔戒语音高光**：现场是否真用麦克风？（需 `localhost`/HTTPS 安全上下文 + 授权，路演网络/权限有风险）。建议**默认走滚动自动点亮**，麦克风作为可选彩蛋。
4. **中文字体**：思源宋/黑体走 **web font**（体积大、投影现场需联网）还是**本地系统字体**（思源黑→苹方/思源宋→宋体兜底，零加载）？路演稳妥建议本地系统字体兜底。
5. **首屏 Hero 轮播**：用现有 skin effect 截帧轮播，还是直接 mount 4 个 effect 轮流跑（更炫但更重）？

| 6 | **能力缺口补齐**（§1 风险点）：给需要自动演示的 effect 补 0..1 进度入口或 `play()`（魔戒点亮、血字反转、爱丽丝坠落、面纱散雾） | 评委不操作也能看懂 |

## 8. 一句话定调

> 不要做成"我们做了四个模块"的说明页，而要做成
> **用户亲自滚动进入四种阅读体验的发布页**：
> 用滚动控制剧情节奏，用核心意象建立文学气质，用一个"神奇文字交互"证明这本书的精神。
